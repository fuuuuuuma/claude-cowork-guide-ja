# 🎁 Claude Cowork で使える Skills 100選

そのまま `.claude/skills/<name>/SKILL.md` として保存して呼び出せる実物のスキル集。各コードブロックの中身がSKILL.md本体です。フォルダに置くか、Coworkの「カスタマイズ → スキル」からアップロードして使えます。

## 1. ファイル・ドキュメント整理

### 1. folder-organizer — フォルダ自動仕分け
説明: 散らかったフォルダを内容で分類し、統一命名にリネームして索引を作るスキル。
```text
---
name: folder-organizer
description: フォルダ内のファイルを内容で分類・サブフォルダ化し、YYYY-MM-DD_内容名 に統一リネームして索引markdownを作りたいとき
---

# フォルダ自動仕分け

1. 指定フォルダの中身を確認し、内容ごとに分類案（サブフォルダ名）を提示する。サブフォルダは掘りすぎない。
2. ユーザーの承認後、各ファイルを「YYYY-MM-DD_内容がわかる短い名前」に統一リネームして該当サブフォルダへ移動する。
3. 破壊的操作（上書き・削除）は必ず事前に一覧を出して承認を得る。
4. 完了後、どのサブフォルダに何が入っているかを一覧した index.md を作る。
```

### 2. duplicate-finder — 重複ファイル検出・整理
説明: 同一・類似内容のファイルをファイル名やサイズ、内容の近さから検出し、残すものを選んで整理するスキル。
```text
---
name: duplicate-finder
description: フォルダ内に重複・類似ファイル（コピー、旧版、同名違いなど）が疑われ、整理・削減したいとき
---

# 重複ファイル検出・整理

1. 対象フォルダを走査し、ファイル名・サイズ・更新日時・内容の近さから重複候補をグループ化する。
2. 各グループについて「最新版と思われるもの」「古い/コピーと思われるもの」を推定し、根拠とともに一覧表を作る。
3. 一覧表をユーザーに提示し、削除・統合してよいグループを確認してもらう。推定に自信がない組はスキップして質問する。
4. 承認された分だけ、削除ではなくまず「重複_隔離」フォルダへ移動する（即削除はしない）。
5. 処理結果（削減件数・削減容量・残した基準）をmarkdownで報告する。
```

### 3. file-renamer — 命名規則統一リネーム
説明: バラバラなファイル名を指定した命名規則（日付・案件名・連番など）に一括変換するスキル。
```text
---
name: file-renamer
description: フォルダ内のファイル名の付け方がバラバラで、日付や案件名などの統一ルールで一括リネームしたいとき
---

# 命名規則統一リネーム

1. ユーザーに希望の命名規則（例: YYYY-MM-DD_案件名_連番.拡張子）を確認する。指定がなければ現状のパターンから推測し提案する。
2. 対象フォルダの現ファイル名を一覧し、新旧対応表（旧名→新名）を作成する。
3. 対応表をユーザーに提示し、承認を得てから実際にリネームを実行する。
4. リネームで名前が衝突する場合は連番を付与するなどのルールをあらかじめ決めておく。
5. 完了後、対応表をrename_log.csvとして保存し、元に戻したいときの参照にできるようにする。
```

### 4. doc-indexer — ドキュメント索引作成
説明: 大量のドキュメント群を読み、各ファイルの内容要約と場所をまとめた索引ページを作るスキル。
```text
---
name: doc-indexer
description: 大量のドキュメント・資料が溜まっていて、内容が一目でわかる索引・目次を作りたいとき
---

# ドキュメント索引作成

1. 対象フォルダ配下のドキュメントを再帰的に確認する（サブフォルダも含む）。
2. 各ファイルについて1〜2行の内容要約と、種類（議事録/契約書/企画書など）を推定する。
3. カテゴリ別・更新日順など見やすい切り口で一覧表を作る。ユーザーに切り口の好みを確認してもよい。
4. index.md として保存し、各項目からファイルへのリンク（相対パス）を付ける。
5. 内容の推定に自信が持てないファイルは「要確認」として明示し、断定しない。
```

### 5. inbox-triage — ダウンロードフォルダ自動仕分け
説明: ダウンロードフォルダに溜まった雑多なファイルを種類別・案件別に仕分けし、不要そうなものを提案するスキル。
```text
---
name: inbox-triage
description: ダウンロードフォルダやデスクトップにファイルが溜まって散らかっており、定期的に仕分け・整理したいとき
---

# ダウンロードフォルダ自動仕分け

1. 対象フォルダの中身を確認し、種類（請求書/資料/画像/インストーラ/一時ファイルなど）を分類する。
2. 案件やプロジェクトに紐づけられるものは、既存の該当フォルダへの移動先を提案する。行き先が不明なものは「未分類」に残す。
3. 明らかに一時的・不要と思われるファイル（重複ダウンロード、古いインストーラ等）は削除候補として一覧化する。削除は必ず事前確認を取り、承認分のみ実行する。
4. 移動・削除の結果をサマリーとして報告する。
5. 定期実行する場合は、同じ分類ルールを次回以降も踏襲できるようメモを残す。
```

### 6. archive-organizer — 過去プロジェクトのアーカイブ整理
説明: 完了したプロジェクトのフォルダを、命名統一・不要ファイル除去のうえアーカイブ用の場所へ移すスキル。
```text
---
name: archive-organizer
description: 完了・凍結したプロジェクトのフォルダを整理してアーカイブ保管したいとき
---

# 過去プロジェクトのアーカイブ整理

1. 対象プロジェクトフォルダの中身を確認し、成果物（納品物・最終版資料）と作業途中ファイル（下書き・一時ファイル）を区別する。
2. 作業途中ファイルのうち明らかに不要なものを削除候補として一覧化し、承認を得る。
3. 成果物フォルダを「YYYY-MM_プロジェクト名」形式に統一リネームする。
4. 指定のアーカイブ先フォルダへ移動し、何を含む・含まないかを記したREADME.mdを添える。
5. 移動が完了したら元の作業場所からの移動結果を一覧で報告する。
```

### 7. naming-convention-auditor — ファイル名規則の監査・修正提案
説明: フォルダ内のファイル名が社内・チームの命名規則に沿っているかを点検し、違反箇所を洗い出すスキル。
```text
---
name: naming-convention-auditor
description: チームやプロジェクトで決めたファイル命名規則が守られているか点検し、違反を洗い出したいとき
---

# ファイル名規則の監査・修正提案

1. ユーザーに命名規則（フォーマット・禁止文字・必須要素など）を確認する。既存の規則ドキュメントがあれば読み込む。
2. 対象フォルダのファイル名を規則と照合し、違反しているものを理由付きで一覧化する（例: 日付の桁不足、全角記号混入など）。
3. 違反ファイルそれぞれについて修正後の名前案を提示する。
4. ユーザーの承認を得た分のみ一括リネームを実行する。
5. 監査結果と修正件数をレポートとしてまとめる。
```

### 8. version-cleanup — 古いバージョンファイルの整理
説明: 「final」「final2」「修正版」など乱立したバージョンファイルを整理し、最新版を明確化するスキル。
```text
---
name: version-cleanup
description: 同じ資料のバージョン違いファイル（final、final2、修正版など）が乱立していて最新版がわかりにくいとき
---

# 古いバージョンファイルの整理

1. 対象フォルダ内で同一資料と思われるファイル群をファイル名・更新日時から特定しグループ化する。
2. 各グループの更新日時・ファイル名の文言（final、修正、v2等）から最新版候補を推定し、根拠を提示する。
3. ユーザーに最新版の確認を取る。判断が割れる場合は中身の差分も確認して報告する。
4. 承認後、最新版のみを分かりやすい名前でメインフォルダに残し、旧版は「旧版」サブフォルダへ移動する（削除はしない）。
5. 整理結果を一覧表で報告する。
```

### 9. receipt-organizer — 領収書・請求書のファイル整理
説明: 集めた領収書・請求書の画像やPDFを日付・取引先ごとに仕分け、統一命名で保存するスキル。
```text
---
name: receipt-organizer
description: 領収書や請求書のファイル・画像が未整理で溜まっており、日付や取引先ごとに整理して保存したいとき
---

# 領収書・請求書のファイル整理

1. 対象フォルダ内の領収書・請求書ファイルを確認し、それぞれの日付・取引先・金額を読み取る（画像・PDFの内容を確認する）。
2. 読み取り内容を一覧表（日付・取引先・金額・ファイル名）として作成し、ユーザーに確認してもらう。読み取りに自信がないものは「要確認」と明示する。
3. 承認後、「YYYY-MM_取引先名_金額」形式にリネームし、月別サブフォルダへ移動する。
4. 一覧表をCSVとしても保存し、経費精算や会計処理に転用できるようにする。
5. 移動・リネームは事前確認を取ってから実行する。
```

### 10. cross-drive-consolidator — 散在資料の統合整理
説明: 複数のフォルダ・場所に散らばった同一プロジェクトの資料を1か所に集約し、重複を整理するスキル。
```text
---
name: cross-drive-consolidator
description: 同じプロジェクトの資料が複数のフォルダに散らばっており、1か所に集約して整理したいとき
---

# 散在資料の統合整理

1. ユーザーに対象プロジェクトに関連しそうなフォルダの場所を確認する（複数指定可）。
2. 各場所を確認し、対象プロジェクトに関連するファイルを洗い出して一覧化する。
3. ファイル名や内容から重複・旧版を検出し、統合後の残し方（どちらを正とするか）を提案する。
4. ユーザーの承認を得てから、統合先フォルダへコピーまたは移動し、統一命名でリネームする。
5. 統合結果として、元の場所と統合先の対応表、および重複として除外したファイルの一覧を報告する。
```

## 2. スライド・レポート整形

### 11. brand-template-applier — ブランドテンプレ適用
説明: 既存資料の内容を維持したまま、指定のブランドテンプレ（色・フォント・ロゴ）に流し込み直すスキル。
```text
---
name: brand-template-applier
description: 作成済みの資料やスライドを、指定のブランドカラー・フォント・ロゴに沿ったテンプレへ体裁だけ変換したいとき
---

# ブランドテンプレ適用

1. ブランドガイドライン（配色・フォント・ロゴファイルの場所）をフォルダ内またはユーザーからの指定で確認する。
2. 変換対象の資料を読み、内容（テキスト・図表・画像）と体裁（色・フォント）を分離して把握する。
3. 内容は変えずに、色・フォント・ロゴ配置のみをブランド仕様に合わせて再構成した版を作る。
4. スライド枚数が多い場合は最初の数枚を先に提示し、方向性の承認を得てから残りを一括処理する。
5. 完成版を別名で保存し、元ファイルは上書きしない。
```

### 12. deck-outline-builder — 構成案からスライド生成
説明: 箇条書きの構成メモや台本から、章立て・1枚あたりの要点を整理したスライド資料を組み立てるスキル。
```text
---
name: deck-outline-builder
description: 箇条書きの構成メモや台本テキストから、章立てされたスライド資料を組み立てたいとき
---

# 構成案からスライド生成

1. 元になる構成メモ・台本を読み、主張の流れ（背景→課題→提案→まとめ等）を把握する。
2. 章立てとスライド枚数の目安を提案し、ユーザーに承認を得る。
3. 各スライドについて「見出し＋要点3つ以内」の形で内容を整理する。詰め込みすぎない。
4. 図表が必要そうな箇所は「図解案」としてテキストで示し、実データがあれば表やグラフの形に落とし込む。
5. 全体をひとつの資料（スライド用HTMLまたは指定フォーマット）にまとめ、通しで確認できる状態で提出する。
```

### 13. report-formatter — レポート体裁統一
説明: 複数人が書いた文書やメモを、見出しレベル・表記ゆれ・フォーマットを統一した1つのレポートに整えるスキル。
```text
---
name: report-formatter
description: 複数のメモや文書をひとつのレポートとして体裁（見出し・表記・フォーマット）を統一してまとめたいとき
---

# レポート体裁統一

1. 対象の文書・メモをすべて読み、内容の重複や矛盾がないか確認する。矛盾があれば先にユーザーへ確認する。
2. 見出しレベル・箇条書き記号・数値の書式（桁区切り、単位）・表記ゆれ（同じ意味の言葉の揺れ）を統一する方針を決める。
3. 方針に沿って全体を1つの文書へ整形し直す。元の主張・数値は変えない。
4. 表紙・目次が必要なら追加する。
5. 完成稿をユーザーに提示し、内容面で変えてほしい箇所がないか確認する。
```

```text
4. どちらの表記・体裁に統一するかユーザーに確認し、承認後に一括修正する。
5. 修正後の資料と、修正箇所一覧を合わせて提出する。
```

### 15. exec-summary-builder — エグゼクティブサマリー抽出

### 19. meeting-recap-deck — 議事内容のスライド化
