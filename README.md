## 1. ClaudeCowork とは — 「パソコンを触れる手が生えたAI」

ChatGPTのようなAIチャットは、質問に「答える」ところまでが仕事でした。ClaudeCoworkは、そこから一歩進んで、**答えるだけでなく実際に手を動かして仕事を終わらせてくれる**のが特徴です。公式サイトの言葉を借りれば、Coworkとは「タスクを渡すと、ファイル・カレンダー・メール・メッセージアプリ・Web、そして自分がつないだ各種ツールを横断して、**仕事が終わるまで自分で動く**」もの。ひとことで言えば「相談相手」ではなく「実際に手を動かしてくれる部下」です。

Coworkは、Claudeの**デスクトップアプリ**（Mac・Windows・ChromeOS・Linux）で使えます。アプリの上部には3つのタブが並んでいて、これがCoworkを理解する最初のポイントです。

- **Chat（チャット）** … いわゆる普通のAIチャット。相談や下書きはできますが、ファイルを操作したりはしません。ChatGPTのような使い心地です。
- **Cowork（コワーク）** … 今回の主役。コーディングの知識がなくても「ほぼ何でも」実行してくれる、ちょうど真ん中の使いやすい場所です。
- **Code（コード）** … 右端にある開発者向けモード。コードを書いたり、ソフトを作ったり、スクリプトを走らせたりします。

大事なのは、**Coworkは「話しかけるだけで実行してくれるチャット」だと思っていい**ということ。見た目はいつものチャット画面と同じで、学ぶのに時間はかかりません。それでいて、使いこなしたときにできることの幅はとても広い——入口はやさしく、天井は高い、という道具です。

2026年7月のアップデートで、Coworkは **web（claude.ai のホームから）とスマホアプリ（iOS/Android）** にも対応しました（[公式ブログ](https://claude.com/blog/cowork-web-mobile/)）。デスクトップは「じっくり深く作業する」主戦場のまま、外出先ではスマホから指示・確認できる、という役割分担になっています。

## 2. なぜ今 Cowork なのか — 3つの理由

### 理由1: 実は「9割超」が開発以外の仕事

Coworkは、開発者向けの Claude Code を土台にして作られています。ここだけ聞くと「じゃあエンジニア向けでしょ？」と思うかもしれません。ところが公式ブログによると、**Cowork利用の9割超は、ソフトウェア開発ではなかった**とのこと。最も多かったのは**ビジネスオペレーション（事務・業務プロセス）とコンテンツ制作**で、この2つだけで利用のおよそ半分を占めています。

たとえば、四半期の支出を突き合わせて差異メモを作る、契約書のフォルダを整理して更新期限とリスクを一覧化する、通話メモとデータから明日の提案資料を組み立てる——といった、**ふつうの知識労働**です。つまり、「AIに任せるのはコードだけ」という時代はもう終わっていて、**大半の人は、エンジニアでなくてもCoworkの恩恵を受けられる**のです。

### 理由2: 今なら上限が2倍、しかもFable 5と相性がいい

Coworkは有料プランで使う機能ですが、**今なら利用上限を2倍にするキャンペーンが2026年8月5日まで**実施されています。始めどきとしては絶好のタイミングです。

さらに、話題のハイエンドモデル **Fable 5** が、2026年7月から Cowork の中でも使えるようになりました（詳しくは第9章）。重い調査や分析、長い文章の読み込みには最強の頭脳を、日常の軽い作業にはお手頃なモデルを——と使い分けられるので、「Fable 5を使うなら、その手足としてCoworkが最適」という関係になっています。

### 理由3: アップデートで、さらに使いやすくなった

前述のとおり、web・スマホ対応で「どこでも作業が続く」ようになりました。加えて、セッション（作業の状態）とファイルがClaudeアカウントに保存され、**デバイスをまたいで持ち歩ける**ように。パソコンを閉じても作業は継続し、**デバイスがオフでもスケジュールした定例タスクが動きます**。判断が必要になったときだけ、スマホに通知が来て承認できる——そんな流れが実現しました。

## 3. 何がすごいのか — Coworkの3本柱

### 柱① 答えるだけでなく「実行」する

Coworkの一番の違いは、**実際にファイルを操作するところ**です。使い方はシンプルで、まず作業させたい**フォルダを選んで「許可」**します。すると、そのフォルダの中身をClaudeが読み、必要なら中身を書き換え、ファイルを移動・リネームし、新しいファイルを作れるようになります。毎回ファイルをアップロードしたり、文脈を説明し直したりする必要はありません。

作業中は、画面の**右側に「計画・手順・開いているファイル・進捗」がリアルタイムで表示**されます。もし方向がずれていると感じたら、**止めて、指示を直して、やり直せます**。そして、削除のような取り返しのつかない操作の前には、**承認を求めてくる**ので安心です。

### 柱② コマンドライン不要 — Codeと「同じ頭脳」を、黒い画面なしで

「Coworkってつまり何がCodeと違うの？」——答えは"入口だけ"です。Anthropicは、CoworkはClaude Codeを使って作られ、同じ「計画する→実行する→自分でチェックする」仕組みを受け継いでいる、と説明しています。違うのは、Codeがターミナル（黒い画面）で動くのに対し、Coworkは**普通のアプリ画面**で動くという点。ターミナルが苦手な人でも、まったく同じ賢さを使えるわけです。

### 柱③ 連携・記憶・自動化で「一度きり」が「仕組み」になる

Coworkが単なる作業代行で終わらないのは、次の3つがあるからです。

- **コネクタ（連携）** … Gmail・Googleカレンダー・Googleドライブ・Slack・CRMなど、**外部のツールとつなぐ**仕組みです（技術的には「MCP」とも呼ばれます）。つなげば、メールを読んで下書きしたり、カレンダーを見て予定を組んだりできます。
- **スキル（型）** … よくやる作業の手順やテンプレートを**1つのパッケージにまとめて、いつでも呼び出せる"型"**にしたものです。一度うまくいったやり方を保存しておけば、次からは同じ品質で再現できます。
- **スケジュール（自動化）** … 毎朝・毎週の定例作業を、**時間を決めて自動で実行**させられます。「毎週月曜の朝8時に、先週比のレポートを作って」と登録すれば、あとは勝手に仕上がります。

## 4. Code と Cowork、どちらを使えばいい？

どちらも同じ頭脳ですが、向いている人と場面が違います。迷ったときの目安は「**コードやgit（開発）が絡むかどうか**」だけです。

| 観点 | Cowork | Claude Code |
|---|---|---|
| 向いてる人 | コマンド不要で使いたい知識労働者 | 開発者・エンジニア |
| 使う場所 | デスクトップ・web・スマホアプリ | ターミナル・IDE |
| 得意なこと | 資料・調査・ファイル整理・業務全般 | コード・git・スクリプト実行 |
| 安全性 | 許可したフォルダ・ツールだけ（隔離環境） | 起動した人の全権限で動く |
| 頭脳 | 同じ（Codeを使って作られている） | 同じ |

Coworkは**サンドボックス（隔離された環境）**で動くため、「フォルダとツールはあなたが選び、Claudeはそれ以外に触れない」「削除には承認が要る」という安全設計になっています。コードを書く仕事なら Code、**それ以外の知識労働なら Cowork でほぼ足りる**——これが基本の使い分けです。

3つのタブの早見表も載せておきます。

| できること | Chat | Cowork | Code |
|---|---|---|---|
| 会話・相談 | ◎ | ◎ | ○ |
| ファイル操作 | × | ◎ | ◎ |
| 外部連携（コネクタ） | △ | ◎ | ◎ |
| コマンド不要 | ◎ | ◎ | × |
| 主な対象 | だれでも | 知識労働者 | 開発者 |

## 6. 使い方 — フォルダを渡して、ゴールを伝えるだけ

ここからは初心者基準で、実際の使い方を見ていきます。難しく考える必要はありません。

### 基本の4ステップ

1. **入れる** … Claudeのデスクトップアプリをダウンロードしてサインイン（有料プラン）。
2. **選ぶ** … 上部の「Cowork」タブを開き、作業させたいフォルダを選んで「許可（または常に許可）」。
3. **伝える** … チャット欄に、**手順ではなく「成果物とゴール」**を書いてお願いする。
4. **受け取る** … 途中の質問に答えたり、大事な操作を承認したりしながら、完成した成果物を受け取る。

### いちばん大事なコツ：「Say what, not how（手順でなく、成果を伝える）」

Coworkに頼むときのコツは、細かい手順を指示しないこと。**「何を・どんな状態にしてほしいか」だけを伝えれば、やり方はClaudeが考えてくれます**。あなたは"監督"、Coworkは"実行してくれるスタッフ"だとイメージしてください。

### こんな悩み → こう頼むだけ

| こんな悩み | こう頼むだけ |
|---|---|
| 資料づくりに時間がかかる | 「このフォルダの資料から、提案用のデッキを作って」 |
| ファイルがぐちゃぐちゃで探せない | 「中身で仕分けして、日付＋内容の統一名にリネームして」 |
| 毎週の集計が手作業でつらい | 「毎週月曜の朝に、先週比でレポートをまとめて」 |
| 議事録づくりが面倒 | 「この文字起こしを、前回のフォーマットで議事録にして」 |

### そのままコピペで試せる例

```
このフォルダの請求書一覧と入出金明細を突き合わせて、
未回収・二重入金・金額不一致をシートに分けてExcelにまとめて。
実行する前に、進め方の方針を見せて。
```

うまくいったら、もう一歩。

```
今のやり方を「請求突合」という名前でスキルにして、
次回から呼び出せるようにして。
```

こうして"型"を貯めていくと、同じ品質の作業をいつでも再現できるようになります。

## 7. もっと使いこなす — スキル・連携・自動化・パーソナライズ

本筋は「フォルダを渡してお願いするだけ」で十分ですが、慣れてきたら次の4つで一気に世界が広がります。いずれも難しい操作は要りません。

### スキル（型）を作る

スキルとは「特定の作業を、あなた好みのやり方でこなすための手順・知識・テンプレを一式にまとめたパッケージ」です。実体は**マークダウンのファイル**で、フォルダに入れて呼び出します。作り方はとても簡単で、**ある作業をやり終えたあとに「今やったことをスキルにして。名前は〇〇」とお願いするだけ**。Coworkには「スキルを作るためのスキル（skill creator）」が入っているので、Claudeが自動でファイルを整え、うまくいくか自己テストまでしてくれます。気に入ったら「copy to your skills」で自分のスキル一覧に登録。以後は文脈に応じて自動で発動します。設定の「カスタマイズ → スキル」から、既存のスキルを有効化したりアップロードしたりもできます。

（技術的な補足：このスキルの形式は Claude Code と共通で、`.claude/skills/<スキル名>/SKILL.md` というフォルダ形式です。今回のプレゼント「Skills 100選」は、この形式でそのまま保存して呼び出せるようにしてあります。）

### コネクタ（外部連携）をつなぐ

「＋」ボタンからコネクタを開くと、Gmail・カレンダー・ドライブ・Slack・CRMなど、たくさんの外部ツールにつなげます。ライブラリをカテゴリや検索で絞り、「＋」で有効化して認証すればOK。つないだあとは、ツールごとに「常に許可／承認が必要／ブロック」と権限を細かく調整できるので、外部への操作も常に自分の管理下に置けます。

### スケジュールで定例を自動化する

左メニューの「スケジュール」から、名前・依頼内容・実行頻度（毎日/毎週など）・使うモデル・フォルダを設定すれば、定例作業が自動で回り始めます。「起きたら重要メールを要約して」「毎月1日にフォルダを整理して」といった"やり続けたいこと"を、デバイスがオフでも実行してくれます。実行履歴も残るので、後から結果を見返せます。

### パーソナライズで"自分向け"にする

設定には2種類の personalization があります。1つは**Claude全体に効く設定**（口調・トーン・書式・情報源の好みなど）、もう1つは**Coworkだけに効く設定**（ファイル名や日付の形式など）。「省略形は使わない」「過度な太字は避ける」「一次情報を優先する」といった好みを一度書いておけば、毎回同じことを指示せずに済みます。

## 8. 職種別ユースケース — あなたの仕事のどこで使うか

「9割超が開発以外」という数字が示すとおり、Coworkは職種を問わず効きます。代表的な例を挙げます。

- **営業** … CRM・通話記録・メール・カレンダーをつなぎ、明日の商談ブリーフ（参加者・直近のやり取り・先方の動き・論点）を自動生成。毎朝スケジュールで受け取ることも。
- **マーケティング・広報** … キャンペーンのデータから、毎週月曜に4枚のデッキと推奨アクションを自動作成。メディア露出レポートを前週比で更新。
- **経理・財務** … 請求書一覧と入出金明細を突き合わせ、未回収・二重入金・金額不一致をシート分け。関数を書かずに照合が終わります。
- **法務・総務** … 契約フォルダを分野別に仕分け・リネームし、抜けている論点（カバレッジgap）を洗い出し。大量ファイルや領収書の整理も。
- **カスタマーサクセス・CS** … 移動中にダッシュボードを作成し、スマホで続きを引き継ぐ。
- **経営企画・リサーチャー** … 複数のソースを横断して調査し、共通テーマや矛盾点を抽出して資料化。
- **クリエイター・情報発信** … 過去投稿を踏まえた投稿案づくり、下書き・編集・整理。

これらを支える発想は共通です——「毎週かかっている時間の長い作業」ほど、Coworkに任せる価値が大きい。プレゼントの「職種別Cowork活用術100選」では、10職種×各10個の具体タスクを、職種で検索・絞り込みできる形にまとめています。

## 9. Fable 5 と Cowork — なぜ話題で、どう相性がいいのか

Fable 5は、2026年に登場したAnthropicのハイエンドモデルです。ほぼすべてのベンチマークで最高水準とされ、ソフトウェア開発だけでなく、知識労働・画像理解・科学研究などで高い性能を示します。この夏「話題」になっているのには理由があります。**6月9日に発表 → 6月に輸出規制の影響で一時停止 → 安全対策を強化して7月1日に世界で復活**、という経緯をたどったからです。

そして復活と同時に、Fable 5は **Claude.ai・Claude Code・そしてCowork** で使えるようになりました（[Anthropic公式](https://www.anthropic.com/news/redeploying-fable-5)）。ここが「Fable 5を使うなら、Coworkで良くない？」という今回のテーマの核心です。**CoworkでモデルをFable 5に切り替えれば、"最強の頭脳"に"パソコンを触れる手"を与えられる**わけです。

とはいえ、いつもFable 5である必要はありません。**重い調査・分析・長文の読み込みにはFable 5、日常の軽い作業にはSonnetで十分**、という使い分けが現実的です（Sonnetは軽くて速く、Opusはより高度な処理向け、Haikuはごく簡単な即答向け）。用途に合わせてモデルを選べるのも、Coworkの強みのひとつです。

## 10. 使いこなす心得 — プロンプトより先に「業務の棚卸し」

最後に、いちばん大切な考え方を。多くの人が最初につまずくのは、「プロンプトの正解」を探しすぎることです。プロンプト集をたくさん集めても、**自分の仕事のどこに当てはめるかを考えていないと、結局一度も使わずに終わります**。

先にやるべきは、**自分が何に時間を使っているかの棚卸し**です。「毎週3時間かかっているレポート作成」「1通のメールに30分悩んでいる」——そうやって書き出すと、「あれ、これAIで良くない？」という作業がいくつも見つかります。そこを1つ選んで、Coworkに任せてみる。これが最短の入口です。

もう1点、安全面の心得も。触れられるのは許可したフォルダだけですが、**機密情報を扱うフォルダは慎重に**。承認を挟みながら使い、コミュニティ製のスキルを取り込むときは中身を必ず確認しましょう（悪意ある指示が混ざっていないか）。
